## 用户输入

```text
$ARGUMENTS
```

你**必须**先考虑用户输入(如果不为空)。

## 任务目标

ultrathink 模式，对目标内容进行多维度交叉检查。

## 输入来源

- 当用户输入不为空时，根据输入内容进行检查
- 当用户输入为空时，优先检查上一轮用户消息涉及的内容；若仍无法确定，提示用户指定检查对象

## Review 要点

根据检查对象（文档、代码、方案、变更、操作等）自动选择合适的 review 角度，参考但不限于：

- **准确性**：是否正确、是否有杜撰猜测
- **完整性**：是否有遗漏、边界是否覆盖
- **一致性**：命名风格是否统一、前后表述是否矛盾；有项目上下文时检查与现有约定是否一致，否则仅做内部一致性检查
- **合理性**：结构是否清晰、逻辑是否通顺、是否有冗余或重复
- **可读性**：表达是否清晰易懂、是否有歧义
- **关联影响**：是否识别了所有关联项并覆盖了受影响的地方
- **安全与风险**：安全隐患、性能风险、异常边界等（仅对代码和技术方案适用）

## 输出格式

按以下格式输出 Review 结果：

```
Review 结果
- 严重问题：X 个
- 一般问题：X 个
- 轻微问题：X 个

=======================================
## 严重问题（X个）- 影响正确性或存在明显错误
=======================================

-- [1.1] 问题类型 - 具体位置（行号/章节/段落）

    描述：问题说明
    建议：修改方案

-- [1.2] ...

=======================================
## 一般问题（X个）- 影响可维护性或存在遗漏
=======================================

-- [2.1] 问题类型 - 具体位置（行号/章节/段落）

    描述：问题说明
    建议：修改方案

-- [2.2] ...

=======================================
## 轻微问题（X个）- 可改善但不影响功能
=======================================

-- [3.1] 问题类型 - 具体位置（行号/章节/段落）

    描述：问题说明
    建议：修改方案

-- [3.2] ...

=======================================
整体结论：...
=======================================
请输入修复指令:
提示: A-全部修复 | S-仅严重 | G-仅一般 | M-仅轻微 | N-跳过 | 自定义方案
```
